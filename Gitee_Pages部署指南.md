## 🚀 部署步骤

### 方法一：从 GitHub 导入（推荐，最简单）

如果你已经有 GitHub 仓库，可以直接导入：

#### 步骤 1：导入仓库

1. 登录 Gitee
2. 点击右上角 **"+"** → **"导入仓库"**
3. 填写信息：
   - **导入源地址**：`https://github.com/RiverXue/MdToEvery.git`
   - **仓库名称**：`MdToEvery`（或自定义）
   - **仓库介绍**：Markdown转换工具 - AI内容分享神器
   - **是否开源**：选择"公开"
4. 点击 **"导入"** 按钮
5. 等待导入完成（通常 1-2 分钟）

#### 步骤 2：启用 Gitee Pages

1. 进入导入的仓库页面
2. 点击顶部菜单 **"服务"** → **"Gitee Pages"**
3. 在 Gitee Pages 设置页面：
   - **部署分支**：选择 `master` 或 `main`（根据你的分支名）
   - **部署目录**：选择 `/`（根目录）
   - **强制使用 HTTPS**：勾选（推荐）
4. 点击 **"启动"** 按钮
5. 等待部署完成（通常 1-3 分钟）

#### 步骤 3：访问你的网站

部署成功后，你的网站地址将是：
```
https://你的用户名.gitee.io/MdToEvery/
```

例如：`https://riverxue.gitee.io/MdToEvery/`

**重要提示**：
- 首次部署需要等待几分钟
- 部署成功后，地址会显示在 Gitee Pages 设置页面
- 可以点击"更新"按钮手动更新部署

### 方法二：手动上传文件

如果你没有 GitHub 仓库，可以手动创建：

#### 步骤 1：创建新仓库

1. 登录 Gitee
2. 点击右上角 **"+"** → **"新建仓库"**
3. 填写信息：
   - **仓库名称**：`MdToEvery`（或自定义）
   - **仓库介绍**：Markdown转换工具 - AI内容分享神器
   - **是否开源**：选择"公开"
   - **初始化仓库**：可以勾选"使用Readme文件初始化这个仓库"
4. 点击 **"创建"** 按钮

#### 步骤 2：上传文件

有两种方式上传文件：

**方式 A：网页上传（适合少量文件）**

1. 进入仓库页面
2. 点击 **"上传文件"** 按钮
3. 拖拽或选择所有项目文件上传
4. 填写提交信息：`初始提交：添加项目文件`
5. 点击 **"提交"** 按钮

**方式 B：使用 Git 命令（推荐，适合开发者）**

```bash
# 1. 克隆仓库（如果已创建）
git clone https://gitee.com/你的用户名/MdToEvery.git
cd MdToEvery

# 2. 复制所有项目文件到仓库目录
# （将你的 index.html, style.css, script.js 等文件复制到这里）

# 3. 添加文件
git add .

# 4. 提交
git commit -m "初始提交：添加项目文件"

# 5. 推送到 Gitee
git push origin master
```

#### 步骤 3：启用 Gitee Pages

1. 进入仓库页面
2. 点击顶部菜单 **"服务"** → **"Gitee Pages"**
3. 在设置页面：
   - **部署分支**：选择 `master`
   - **部署目录**：选择 `/`（根目录）
   - **强制使用 HTTPS**：勾选
4. 点击 **"启动"** 按钮
5. 等待部署完成

## 🔄 更新网站内容

### 方法一：网页更新（简单）

1. 在 Gitee 仓库页面，直接编辑文件
2. 或上传新文件替换旧文件
3. 提交更改后，进入 **"服务"** → **"Gitee Pages"**

### 方法二：Git 命令更新（推荐）

```bash
# 1. 进入项目目录
cd MdToEvery

# 2. 修改文件后，添加更改
git add .

# 3. 提交更改
git commit -m "更新描述"

# 4. 推送到 Gitee
git push origin master

# 5. 在 Gitee 网页上，进入"服务" → "Gitee Pages" → 点击"更新"
```

**注意**：Gitee Pages 不会自动部署，需要手动点击"更新"按钮。

## 📝 重要说明

### 1. 部署分支

- 如果从 GitHub 导入，分支可能是 `main`
- 如果是新建仓库，默认分支是 `master`
- 在 Gitee Pages 设置中选择正确的分支

### 2. 部署目录

- 选择 `/`（根目录）表示部署整个仓库
- 如果你的 `index.html` 在根目录，选择 `/` 即可

### 3. 手动更新

- **重要**：Gitee Pages 不会自动部署
- 每次推送代码后，需要手动进入 **"服务"** → **"Gitee Pages"** → 点击 **"更新"** 按钮
- 或者设置自动更新（需要 Gitee 会员）

### 4. 访问地址格式

```
https://你的用户名.gitee.io/仓库名/
```

例如：
- 用户名：`riverxue`
- 仓库名：`MdToEvery`
- 访问地址：`https://riverxue.gitee.io/MdToEvery/`

### 5. 自定义域名（可选）

如果需要使用自己的域名：

1. 在 Gitee Pages 设置中，填写自定义域名
2. 在域名 DNS 设置中添加 CNAME 记录
3. 指向：`你的用户名.gitee.io`

## ❓ 常见问题

### Q: 部署后显示 404 错误？

**A:** 检查以下几点：
1. 确保 `index.html` 在仓库根目录
2. 检查 Gitee Pages 设置中的"部署目录"是否为 `/`
3. 确保选择了正确的分支（`master` 或 `main`）
4. 等待几分钟后刷新（首次部署需要时间）

### Q: 样式或脚本不加载？

**A:** 检查以下几点：
1. 确保所有文件都已上传到仓库
2. 检查文件路径是否正确（使用相对路径）
3. 清除浏览器缓存后重试
4. 检查浏览器控制台是否有错误信息

### Q: 如何查看部署状态？

**A:** 
1. 进入仓库页面
2. 点击 **"服务"** → **"Gitee Pages"**
3. 可以看到部署状态和访问地址

### Q: 部署需要多长时间？

**A:**
- 首次部署：1-3 分钟
- 手动更新：1-2 分钟

### Q: 为什么需要手动更新？

**A:** 
- Gitee Pages 免费版需要手动更新
- 这是 Gitee 的限制，与 GitHub Pages 不同
- 如果需要自动更新，需要 Gitee 会员（付费）

### Q: 可以同时部署到 GitHub 和 Gitee 吗？

**A:** 可以！
- GitHub Pages：国际用户访问
- Gitee Pages：国内用户访问
- 在 README 中提供两个地址即可
