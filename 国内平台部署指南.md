## 方案一：Vercel（最推荐）⭐

### 为什么选择 Vercel？

- ✅ **国内访问速度快**：有国内 CDN 节点
- ✅ **自动部署**：连接 GitHub 后自动部署
- ✅ **完全免费**：个人项目免费使用
- ✅ **操作简单**：几分钟即可完成
- ✅ **支持自定义域名**：可以绑定自己的域名

### 部署步骤

#### 1. 注册 Vercel 账号

1. 访问：https://vercel.com
2. 点击右上角 **"Sign Up"**（注册）
3. 选择 **"Continue with GitHub"**（使用 GitHub 登录）
4. 授权 Vercel 访问你的 GitHub 账号

#### 2. 导入项目

1. 登录 Vercel 后，点击 **"Add New..."** → **"Project"**
2. 在项目列表中找到你的仓库：`RiverXue/MdToEvery`
3. 点击 **"Import"**（导入）

#### 3. 配置项目（通常使用默认即可）

- **Framework Preset**：选择 `Other` 或 `Vite`（纯静态项目）
- **Root Directory**：`./`（根目录）
- **Build Command**：留空（纯静态项目不需要构建）
- **Output Directory**：留空或填写 `./`

#### 4. 部署

1. 点击 **"Deploy"**（部署）按钮
2. 等待部署完成（通常 1-2 分钟）
3. 部署成功后，会显示访问地址

#### 5. 获取访问地址

部署完成后，你会得到一个类似这样的地址：
```
https://md-to-every.vercel.app
```

或者：
```
https://md-to-every-你的用户名.vercel.app
```

### 自动更新

- ✅ **自动部署**：每次你推送代码到 GitHub，Vercel 会自动重新部署
- ✅ **无需手动操作**：完全自动化

### 自定义域名（可选）

1. 在项目设置中，进入 **"Domains"**（域名）
2. 添加你的自定义域名
3. 按照提示配置 DNS 记录

## 方案二：Netlify（备选）⭐

### 为什么选择 Netlify？

- ✅ **全球 CDN**：访问速度快
- ✅ **自动部署**：连接 GitHub 后自动部署
- ✅ **完全免费**：个人项目免费使用
- ✅ **功能强大**：支持多种功能

### 部署步骤

#### 1. 注册 Netlify 账号

1. 访问：https://www.netlify.com
2. 点击右上角 **"Sign up"**（注册）
3. 选择 **"GitHub"** 登录
4. 授权 Netlify 访问你的 GitHub 账号

#### 2. 导入项目

1. 登录后，点击 **"Add new site"** → **"Import an existing project"**
2. 选择 **"Deploy with GitHub"**
3. 在仓库列表中找到 `RiverXue/MdToEvery`
4. 点击仓库名称

#### 3. 配置项目

- **Branch to deploy**：`main` 或 `master`（根据你的分支）
- **Build command**：留空（纯静态项目）
- **Publish directory**：留空或填写 `./`

#### 4. 部署

1. 点击 **"Deploy site"**（部署站点）
2. 等待部署完成（通常 1-2 分钟）
3. 部署成功后，会显示访问地址

#### 5. 获取访问地址

部署完成后，你会得到一个类似这样的地址：
```
https://random-name-123456.netlify.app
```

### 自动更新

- ✅ **自动部署**：每次推送代码到 GitHub，Netlify 会自动重新部署

## 方案三：Coding Pages（腾讯云）⭐

### 为什么选择 Coding Pages？

- ✅ **国内访问速度快**：服务器在国内
- ✅ **完全免费**：个人用户免费
- ✅ **腾讯云支持**：稳定可靠

### 部署步骤

#### 1. 注册 Coding 账号

1. 访问：https://coding.net
2. 点击右上角"注册"
3. 填写信息完成注册

#### 2. 创建项目

1. 登录后，点击 **"创建项目"**
2. 选择 **"导入外部仓库"**
3. 输入 GitHub 仓库地址：`https://github.com/RiverXue/MdToEvery.git`
4. 或手动上传文件

#### 3. 启用 Pages 服务

1. 进入项目页面
2. 点击左侧菜单 **"持续部署"** → **"静态网站"**
3. 点击 **"立即创建"**
4. 配置：
   - **构建分支**：选择 `main` 或 `master`
   - **构建目录**：`/`（根目录）
5. 点击 **"保存"**

#### 4. 部署

1. 点击 **"立即部署"**
2. 等待部署完成
3. 获取访问地址

### 更新网站

- ⚠️ **需要手动触发**：修改代码后需要手动点击"重新部署"

## 方案四：本地使用（最简单）⭐⭐⭐⭐⭐

### 为什么选择本地使用？

- ✅ **完全不受网络限制**
- ✅ **速度最快**
- ✅ **无需部署**
- ✅ **隐私最安全**

### 使用方法

1. **下载项目**
   ```bash
   git clone https://github.com/RiverXue/MdToEvery.git
   # 或下载 ZIP 文件解压
   ```

2. **打开文件**
   - 双击 `index.html` 文件
   - 在浏览器中打开即可使用

3. **注意事项**
   - 如果 CDN 资源加载失败，可以使用代理访问一次
   - 或下载库文件到本地（参考下方说明）

### 离线使用（完全本地化）

如果需要完全离线使用，可以下载 CDN 库到本地：

1. **下载库文件**
   - 访问：https://unpkg.com/marked@11.1.1/marked.min.js
   - 访问：https://unpkg.com/docx@8.5.0/build/index.js
   - 访问：https://unpkg.com/file-saver@2.0.5/dist/FileSaver.min.js

2. **创建 lib 目录**
   ```bash
   mkdir lib
   ```

3. **保存文件**
   - 将下载的文件保存到 `lib/` 目录

4. **修改 index.html**
   ```html
   <!-- 替换为本地路径 -->
   <script src="lib/marked.min.js"></script>
   <script src="lib/docx.js"></script>
   <script src="lib/FileSaver.min.js"></script>
   ```

## ❓ 常见问题

### Q: Vercel 在国内访问速度如何？

**A:** Vercel 有国内 CDN 节点，访问速度较快，比 GitHub Pages 快很多。

### Q: 需要付费吗？

**A:** 个人项目完全免费，有充足的免费额度。

### Q: 可以同时部署到多个平台吗？

**A:** 可以！建议同时部署到：
- GitHub Pages（国际用户）
- Vercel（国内用户）
- 本地使用（离线用户）

### Q: 如何更新网站？

**A:** 
- **Vercel/Netlify**：自动更新，推送代码即可
- **Coding Pages**：需要手动触发部署
- **本地使用**：直接修改文件即可

### Q: 部署后 CDN 资源加载失败怎么办？

**A:** 
- 项目已优化为 `unpkg.com` CDN，国内访问较快
- 如果还是失败，可以使用本地库文件（方案四）
